# Real-audio DSP corpus (dev-only)

The "run StrumSight on real music and see the bugs" harness (round 176). It
streams real audio through the **exact shipping DSP** (`LivePipeline` +
`ClipAnalyzer`) off-device, so we can measure detection quality on real audio
**between** physical-guitar APK tests — the gap that let the "chords jump on
speech" bug ship unnoticed.

Nothing here is bundled in the app. `ml/corpus/` is gitignored (audio + report).

## Run the probe

```bash
# Prints a per-clip report (chordShown%, jumpiness, strums, confidence dist)
# and a (rise,release) gate sweep. Writes ml/corpus/report.json.
DSP_PROBE=1 ~/flutter/bin/flutter test test/tools/real_audio_probe_test.dart
```

Without `DSP_PROBE=1` the harness is **skipped** (it is far too heavy for the
normal suite — the sweep alone is 64 full-DSP passes).

## Audio sources

Auto-discovered by the probe:

- `ml/corpus/wav/*.wav` — the dev corpus. Includes deterministic **voice
  negatives** generated by `make_voice_negatives.py` (formant speech, a second
  speaker, a sustained hum, pink noise) — the stimuli that reproduce the
  voice-rejection bug reproducibly.
- `ml/data/klangio/recording_*_phone.wav` — the 82 **real guitar** phone
  recordings (the positive set the fix must not over-reject).

Regenerate the voice negatives:

```bash
python3 ml/corpus/make_voice_negatives.py    # needs numpy (system python3 has it)
```

## YouTube / real songs (`fetch.sh`)

`fetch.sh` downloads short clips with `yt-dlp` → 16 kHz mono WAV. Guitar-**lesson**
clips are ideal test material: they interleave *talking* and *playing*, so one
clip exercises both speech-rejection and chord detection.

**Known limitation (measured 2026-07-14):** YouTube **bot-walls this datacenter
IP** for media downloads ("Sign in to confirm you're not a bot") — metadata
works, audio does not. `fetch.sh` works from a residential IP (e.g. run it on
the phone / a home machine and copy the WAVs over). This is also why in-app
YouTube import is the wrong design: downloading YouTube audio violates YouTube
ToS + app-store policy and would need the client's own network anyway. The
shippable in-app path is "import an audio file you own", not a YouTube URL.
